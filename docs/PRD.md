# AI 职业定位报告 - 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
AI 职业定位报告

### 1.2 产品定位
面向社保局的职业咨询辅助工具。当有人前来咨询职业定位时，社保局老师使用本系统：先输入来访者的基本信息，AI 与来访者进行几轮访谈对话，然后自动生成一份专业的职业定位分析报告，老师可以看着报告为来访者做咨询。

### 1.3 目标用户
- **直接使用者**：社保局职业咨询老师
- **最终受益者**：前来咨询职业定位的来访者
- **购买决策者**：社保局采购方（本产品需进行演示）

### 1.4 核心价值
- 将职业咨询从"凭经验"升级为"有数据、有报告、有依据"
- 缩短咨询准备时间：AI 自动完成信息采集和报告生成
- 提升咨询专业度：标准化的报告格式和可视化图表

---

## 2. 用户流程（4步）

```
步骤1：基本信息填写          步骤2：AI 智能访谈
┌─────────────────────┐    ┌─────────────────────┐
│  老师输入来访者的      │    │  AI 根据基本信息      │
│  岗位、行业等6项信息   │ →  │  自动生成3-5个问题    │
│  （下拉选择为主）      │    │  与来访者进行对话访谈  │
└─────────────────────┘    └─────────────────────┘
                                      ↓
步骤4：查看报告             步骤3：等待动画
┌─────────────────────┐    ┌─────────────────────┐
│  展示完整的职业定位    │    │  8步渐进式动画        │
│  分析报告（图表+文字） │ ←  │  缓解等待焦虑         │
│  老师据此做咨询       │    │  后台同时生成报告      │
└─────────────────────┘    └─────────────────────┘
```

---

## 3. 步骤1：基本信息填写

### 3.1 输入字段

| 序号 | 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| 1 | 岗位名称 | 文本输入 | 是 | 来访者当前或目标岗位，如"产品经理" |
| 2 | 所属行业 | 可搜索下拉 | 是 | 行业列表见附录A |
| 3 | 企业性质 | 下拉选择 | 是 | 民营/国有/外资/合资/上市/创业/政府机关及事业单位 |
| 4 | 城市等级 | 单选 | 是 | 一线/新一线/二线/三线及以下 |
| 5 | 匹配职级 | 可搜索下拉 | 是 | 职级列表见附录B |
| 6 | 工作年限 | 下拉选择 | 是 | 1年以下/1-3年/3-5年/5-10年/10年以上 |

> **与参考产品的差异**：将"企业规模"替换为"工作年限"，因为在职业咨询场景中，来访者的工作年限对职业定位更有参考价值。

### 3.2 交互要求
- 表单居中展示，专业简洁
- 每个字段有清晰的标签和占位提示
- 提交前做必填校验，失败时高亮提示
- 点击"开始职业分析"按钮进入下一步

---

## 4. 步骤2：AI 智能访谈

### 4.1 访谈目的
通过 3-5 轮对话，收集来访者的关键信息，为报告生成提供个性化依据。

### 4.2 访谈问题方向（由 AI 根据基本信息动态生成）

| 方向 | 示例问题 | 信息用途 |
|------|---------|---------|
| 工作经历 | "请简单描述一下您目前的工作内容和主要职责" | 判断岗位匹配度 |
| 核心能力 | "您觉得自己最擅长的2-3项专业能力是什么？" | 能力画像分析 |
| 职业困惑 | "目前在职业发展上最让您困惑或焦虑的是什么？" | 针对性建议 |
| 职业期望 | "未来2-3年，您希望往哪个方向发展？" | 发展路径规划 |
| 行业认知 | "您觉得您所在行业目前面临的最大变化是什么？" | 行业洞察补充 |

### 4.3 访谈交互设计
- 界面：类似微信对话的聊天界面
- AI 消息：左侧气泡，带"AI助手"头像
- 用户回答：右侧气泡
- AI 提问使用流式输出（打字机效果）
- 每轮问答后，AI 自动追问下一个问题
- 3-5 轮后，AI 发出结束语："感谢您的分享，正在为您生成职业定位报告..."
- 底部显示访谈进度（如"第2/4个问题"）

### 4.4 技术要求
- 使用 MiniMax API 生成访谈问题
- 问题需根据用户的基本信息和前序回答动态调整
- 所有问答数据需暂存，用于后续报告生成

## 5. 步骤3：等待动画

### 5.1 设计目标
- 缓解等待焦虑，让用户感到系统在"高速处理"
- 动画时长约 30-50 秒，与实际 AI 生成时间大致匹配
- 如果 AI 提前返回，动画加速完成；如果 AI 较慢，动画适当延长

### 5.2 动画步骤（8步）

| 步骤 | 文案 | 预设时长 | 特效 |
|------|------|---------|------|
| 1 | 连接职业知识库 | 3秒 | 无 |
| 2 | 识别岗位基础信息 | 4秒 | 无 |
| 3 | 分析行业与企业特征 | 5秒 | 无 |
| 4 | 匹配岗位对标数据 | 6秒 | 数字滚动（处理XX万条数据） |
| 5 | 构建岗职价值模型 | 5秒 | 无 |
| 6 | 生成能力画像分析 | 6秒 | 无 |
| 7 | 生成发展路径建议 | 5秒 | 无 |
| 8 | 输出最终报告 | 4秒 | 无 |

### 5.3 视觉设计
- 全屏深色半透明遮罩
- 中央白色面板显示步骤列表
- 当前步骤：蓝色高亮 + 旋转加载圈
- 已完成步骤：绿色背景 + 对勾图标
- 未开始步骤：灰色
- 底部进度条从 0% 推进到 100%
- 底部文案："正在为您生成职业定位报告，请稍候..."

## 6. 步骤4：职业定位报告

### 6.1 报告整体要求
- 单页滚动长报告
- 图文并茂，不只是文字堆叠
- 专业、可视化、有分析价值
- 适合老师打印或展示给来访者看

### 6.2 报告模块（7大模块）

#### 模块1：报告摘要
- 一段话总结来访者的职业定位
- 包含：岗位定位 + 核心优势 + 关键建议
- 放在报告最顶部，让老师快速了解全貌

#### 模块2：岗位概况分析
- 该岗位在当前行业中的定位和价值
- 岗位核心职责（3-6项），用图形化展示（环形图/占比条）
- 岗位在企业中的角色定位（偏执行/管理/协同/规划）
- 该职级在行业中的典型要求

#### 模块3：行业情况分析
- 所在行业的发展趋势（简要）
- 行业对该岗位的需求变化
- 行业内该岗位的竞争态势
- 关键行业洞察（2-3条，带数据支撑）

#### 模块4：能力画像评估
- 雷达图：展示 5-6 项核心能力维度及评估分数
- 每项能力配简短说明
- 标注哪些是核心能力、哪些需要提升
- 结合访谈中来访者提到的能力做个性化分析

#### 模块5：发展路径规划
- 时间线/路径图展示
- 当前岗位的上升通道（2-3个方向）
- 横向迁移可能（1-2个方向）
- 每个方向需要补强的能力
- 短期（1年内）和中期（3年内）的具体建议

#### 模块6：个性化建议
- 基于访谈内容的针对性建议（3-5条）
- 针对来访者提到的职业困惑给出回应
- 具体可操作的行动建议

#### 模块7：补充信息
- 对标岗位参考（同行业同职级的典型岗位）
- 薪酬参考范围（小型辅助区块，不喧宾夺主）
- 数据来源说明
- 免责声明

### 6.3 报告视觉要求
- 主色调：蓝色系（专业商务）
- 浅色背景，深色文字
- 图表配色统一协调
- 模块间有清晰的分隔
- 报告头部显示：报告名称 + 生成日期 + 来访者基本信息

## 7. 设计规范

### 7.1 整体风格
- **风格**：专业商务，简洁大气
- **色调**：浅色系，蓝色为主色
- **字体**：系统默认中文字体（PingFang SC / Microsoft YaHei）
- **暂不放 logo**

### 7.2 页面列表
1. 首页/欢迎页（含品牌标题 + "开始分析"入口）
2. 信息填写页
3. AI 访谈页
4. 等待动画（覆盖层）
5. 报告展示页

### 7.3 响应式要求
- 优先适配桌面端（社保局主要用电脑）
- 平板端基本可用
- 手机端暂不要求

---

## 8. 技术约束

| 项目 | 选择 |
|------|------|
| AI 模型 | MiniMax API（已有 Key） |
| 前端框架 | Next.js + Tailwind CSS + shadcn/ui |
| 图表库 | Recharts |
| 动画库 | Framer Motion |
| 部署方式 | 先本地运行（npm run dev） |

---

## 9. 第一版验收标准

1. 表单页支持 6 个字段输入与校验
2. AI 访谈可以进行 3-5 轮对话
3. 等待动画 8 步完整展示，与后端并行
4. 报告页完整展示 7 大模块
5. 报告中至少有 3 种图表（雷达图、环形图、时间线/路径图）
6. 页面整体视觉专业大气
7. 端到端流程可完整跑通
