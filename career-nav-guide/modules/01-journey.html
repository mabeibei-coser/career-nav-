这个 App 在解决什么问题？

想象你是黄浦区就业中心的咨询员。每天要接待几十个求职者——应届生、刚离职的年轻人、还有年过 35 的资深职场人。每个人的情况都不同，但你手头只有 30 分钟。

这个 App 就是那个「24 小时在线的咨询员」——用户填表、答题、语音访谈，然后拿到一份个性化的职业评估报告。

1
填表

选身份（应届/失业）、上传简历、填学历和工作年限

2
量表测评

8 道情境判断题，测量 4 个性格维度

3
语音访谈

AI 动态生成 4 个问题，用语音回答

4
AI 生成报告

5 个模块：总评、优势、职业定位、简历快诊、行动建议

页面是怎么连起来的？

Next.js 用一种很聪明的方式管理页面——文件夹名就是网址。你只要看文件夹结构，就知道用户能访问哪些页面。

app/ 所有页面都在这里
page.tsx 👉 / 首页（填表）
preparing/ 👉 /preparing 过渡页
quiz/ 👉 /quiz 量表测评
interview/ 👉 /interview 语音访谈
loading/ 👉 /loading 报告生成中
report/ 👉 /report 最终报告
api/ 🔧 后端接口（不是页面）
💡
这叫「约定优于配置」

传统框架需要你手动写路由表（"网址 /quiz 对应这个文件"）。Next.js 反过来：文件放对了位置，路由自动就有了。这是现代 Web 框架最常见的设计哲学之一。

数据是怎么在页面之间传递的？

用户从首页跳到量表页，再跳到访谈页……这几个页面没有直接的「对话」，那表单数据、简历内容是怎么传过去的？

答案是一块公共白板：sessionStorage。

const payload: JobFormData = {

🔑
为什么不用数据库存这些数据？

这些数据只在本次会话里有用，关掉浏览器就没价值了。sessionStorage 比数据库快得多，也不需要服务器。用「最简单的工具解决问题」就是好工程。

表单是怎么检查你有没有填对？

你有没有遇到过：忘填必填项，点提交，页面出现红色报错？这背后是 Zod 在工作。

CODE 

const formSchema = z.object({
  identity: z.enum([
    "recent_grad",
    "young_unemployed",
    "general_unemployed"
  ], { error: "请选择当前身份" }),
  targetPosition: z.string().max(60).optional(),
  education: z.string().min(1, "请选择最高学历"),
});
白话翻译

定义表单「规则书」——里面每条都是一个验证规则

身份字段必须是以下三个值之一：

应届毕业生

35岁以下求职者

35岁及以上求职者

填错了就报错「请选择当前身份」

目标岗位可以不填，但最多 60 个字

学历必填，空着就报「请选择最高学历」
