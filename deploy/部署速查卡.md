## 🔧 常用运维命令（在服务器上）

**部署新版本（最常用）：**
```bash
cd /var/www/career-report
git pull
npm run build
pm2 restart career-report --update-env
```

**看实时日志：**
```bash
pm2 logs career-report
pm2 logs career-report --err     # 只看错误
```

**进程操作：**
```bash
pm2 status
pm2 restart career-report        # 仅重启，不刷新 env
pm2 stop career-report
```

**改管理员密码：**
```bash
cd /var/www/career-report
node scripts/init-admin.mjs      # 按提示输入新密码
vim .env.local                   # 把输出的两行替换 ADMIN_PASSWORD_HASH / ADMIN_SESSION_PASSWORD
pm2 restart career-report --update-env   # ⚠️ 必须 --update-env
```

## 🛡️ 防火墙双层（重要！）

外网 → **腾讯云防火墙** → **服务器 ufw** → 应用，**两层都得放行**：

| 端口 | 用途 | 腾讯云 | ufw |
|---|---|---|---|
| 22 | SSH | ✅ | ✅ |
| 80 | HTTP | ✅ | ✅ |
| 443 | HTTPS（待用） | ✅ | ✅ |
| 3000 | Next.js 直连备用 | ✅ | ✅ |

**云防火墙：** 腾讯云控制台 → 轻量应用服务器 → 实例 → 防火墙
**ufw 操作（在服务器）：**
```bash
ufw status numbered                                  # 查看
ufw allow 80/tcp comment 'HTTP nginx'                # 加规则
ufw delete 4                                         # 按编号删
```

## 🚨 紧急排查清单（网址打不开时按顺序查）

1. **本地代理**：关掉 Clash/V2Ray（端口 7890 那个）再试 — 它返回的 502 假象很多
2. **应用进程**：`ssh ... pm2 status` 看 career-report 是不是 online
3. **直连 3000**：浏览器访问 `http://124.222.114.47:3000` — 通则应用没问题，是 nginx/防火墙
4. **服务器 ufw**：`ssh ... ufw status` 看 80 在不在
5. **腾讯云防火墙**：控制台看 80 入站规则在不在
6. **nginx**：`ssh ... systemctl status nginx`
7. **DNS**：`dig +short h100.jsai100.com` 应该返回 `124.222.114.47`

**504/超时排错口诀：**
> 内通外不通 → 防火墙（腾讯云 OR ufw）
> 内不通 → 应用挂了（pm2/nginx）
> 全通但 502 → 本地代理在捣乱

## 🐛 已踩过的坑（别再踩）

| 坑 | 现象 | 解法 |
|---|---|---|
| bcrypt hash 含 `$` 被 dotenv 吃掉 | 密码总是不对 | hash 必须 base64 编码再存 `.env.local` |
| iron-session `Secure` cookie | 登录 200 但没存 cookie | `ADMIN_COOKIE_SECURE=false` |
| Edge Runtime 模块顶层读 env | curl 通浏览器登不进 | env 读取移到 handler 函数内 |
| ufw 没放 80 | 网页超时但 SSH 通 | `ufw allow 80/tcp` |
| PM2 不刷新 env | 改完 `.env.local` 没生效 | restart 必须加 `--update-env` |
